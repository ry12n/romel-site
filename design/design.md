# ROMEL Landing Page — Design Record

**Version:** V1
**Date:** 2026-06-22
**Stack:** Plain HTML/CSS, single `index.html`
**Deploy target:** GitHub Pages

## Direction
A ROMEL product landing sells through restraint. One clear statement, one access-style CTA, and a roster shown with spec-sheet calm. No SaaS tropes.

## Sections
1. **Hero** — brand statement + primary CTA
2. **Roster** — 10 agents as a personnel grid
3. **Command Layer** — 3 product specs
4. **Access** — single-field request form
5. **Footer** — minimal legal / identity

## Brand application
- Palette: navy-black ground (#080C12), steel-blue accent (#3D8EF0), cold grays
- Typography: Inter for UI, JetBrains Mono for IDs / labels / metadata
- Shape: zero border radius, 1px borders, tonal layering only
- Mood: classified, precise, pre-authentication tension

## Prototype notes
- Form submission is simulated for the prototype; Tom wires the real endpoint.
